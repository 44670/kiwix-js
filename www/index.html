Evopedia : offline wikipedia viewer


Official site : http://www.evopedia.info/

You can search among the article titles, and read any of them without any Internet access.
All the content of wikipedia is inside your device (except the images). It also works with Wiktionary and Wikivoyage.
If your Internet access is expensive/rare/slow/unreliable/watched/censored, you still can browse this amazing amount of knowledge and culture.

NB : If you're connected to the Internet when using the application, the images of the articles will be displayed from the Internet.

Usage

Step 1 : download an archive

From http://dumpathome.evopedia.info/dumps/finished, with a regular computer.
If you get a compressed archive (.zip, .tar.gz or .tar.bz2), you will have to decompress it.
For a quick test, you can start with the dump called "small" : it contains a very small subset of the English archive (note that the hyperlinks between articles will rarely work, as a lot of articles are missing in it).
After that, you can download an archive in your language, through a Bittorrent client. Check its size : these archives take several GB each.

Step 2 : put it on your SD-card

The archive can be stored is any subdirectory of your SD-card (ex: /wikipedia_small_2010-08-14, or /evopedia/wikipedia_small_2010-08-14). If you have enough space, you can put several archives

Step 3 : enjoy wikipedia offline!

Privacy policy

This application works offline. There is no central server. We do not collect any of your personal data and don't even know what you are doing with this application.
Anyway, if you think your Internet access can be watched and/or if you are paranoid, you should shut down your 3G and Wifi access before using the application. If not, the application will try to read the images of the articles on the wikipedia servers, which might indicate which articles you are reading.
This application only reads the wikipedia archives of your SD-card : it does not read any other files.

Feedback / helping / contributing

This is a first version of Evopedia. There are many ways this could be enhanced. In particular :
  • Issue #9 : Some features of the original application still have to be implemented : in particular the "Articles nearby" feature (geolocation)
  • Issue #13 : All the user interface is in English : I plan to translate it at least in French. If you feel able to translate it in your language, please contact me by email
  • Issue #4 : The performance when reading an article can probably be improved. I'm working on it
Suggestions and patches are welcome : the source code is on github. The bugtracker is on github too. I use it as my roadmap.
Alternatively, you can send your feedback by email.
In all cases, please give me as much detail as you can : which device you are using, which version of Evopedia, which archive, which articles etc.

If you like the Evopedia project and use it, please consider giving donations :
  • to the Wikimedia foundation, to help maintaining their servers and article contents. See the donation page
  • to Evopedia, to help maintaining the archives and applications. You can send donations with Flattr or Bitcoin
You can also help by :
  • Adding articles or improving the existing ones, by editing online wikipedia articles
  • Using the ressources of your computer to help generating the archives

Other platforms/versions

Other Evopedia clients exist on various platforms. See the official site : http://www.evopedia.info/

This HTML5/javascript application is currently mainly targeted on Firefox OS devices. But it also can be used on some other platforms :
  • It works perfectly on the Firefos OS simulator (a Firefox plugin), you have to put the archive files in any subdirectory in a "fake-sdcard" folder of your firefox profile (ex : ~/.mozilla/firefox/xxxx.default/extensions/r2d2b2g@mozilla.org/profile/fake-sdcard/wikipedia_small_2010-08-14)
  • On desktops, it works but is less user friendly : you have to manually select the files of your archive, and it's not yet easily deployable as a "web application". I tested it on recent Firefox and Chromium, but it probably works on some other browsers (it even seems to work on IE10). You can test on this online page or from a local copy of the source code
  • On Android, it does not seem to work for now (tried with internal browser, Chrome and Firefox) because you can not select all the files of an archive
  • On Android and desktops, the DeviceStorage API (which is used to browse the SD-card for archives) should become available at some point (see https://wiki.mozilla.org/index.php?title=WebAPI). As soon as it is, I will update the application to make it available on these platforms
